### 1. 문법적 계보: C++의 확장

CUDA(Compute Unified Device Architecture)는 기본적으로 **C++ 문법을 기반으로 설계**되었습니다.

- `.cu` 파일 내에서는 C의 구조체뿐만 아니라 C++의 핵심 기능인 **Template, Overloading, Class** 등을 사용할 수 있습니다.
    
- 실제로 NVIDIA의 CUDA 컴파일러인 `nvcc`는 내부적으로 C++ 컴파일러(g++, cl.exe 등)를 호출하여 호스트 코드를 처리합니다.
    

### 2. 왜 .c나 .cpp를 쓰지 않고 .cu를 쓸까?

단순히 C++ 파일임을 나타내는 것이 아니라, 이 파일 안에 GPU에서 실행될 코드([[GPU Kernel]])가 포함되어 있음을 알리기 위해서입니다.

- **분리 컴파일:** `nvcc`는 `.cu` 파일을 읽어서 `<<< >>>` 같은 CUDA 전용 문법(커널 호출)을 해석합니다.
    
- **코드 분할:** GPU용 코드(Device code)는 NVIDIA 컴파일러가 처리하고, 나머지 CPU용 코드(Host code)는 일반적인 C++ 컴파일러가 처리하도록 분리하는 역할을 합니다.
    

### 3. 실무적 차이

- **`.c`**: 순수 C 언어 문법만 지원 (절차 지향)
    
- **`.cpp`**: C++ 문법 지원 (객체 지향, 템플릿 등)
    
- **`.cu`**: **C++ 문법 + CUDA 전용 키워드**(`__global__`, `__device__` 등) 및 문법 지원
    

---

**요약하자면:**

`.cu`는 **C++을 기반으로 하되, GPU 연산 기능을 추가한 특수 확장자**입니다.
